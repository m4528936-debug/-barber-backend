# 🚀 دیپلوی روی Railway — فقط با مرورگر (بدون نیاز به سیستم/ترمینال)

این راهنما برای کسی نوشته شده که سیستم شخصی نداره و همه‌چیز رو از موبایل یا هر مرورگری انجام می‌ده.

## ✅ خلاصه خیلی سریع (۵ قدم)
1. پروژه رو (از zip اکسترکت‌شده) توی یه ریپوی گیت‌هاب آپلود کن
2. توی Railway.app با گیت‌هاب لاگین کن → New Project → Deploy from GitHub repo
3. یه Volume با مسیر `/app/data` اضافه کن (برای اینکه دیتابیس پاک نشه)
4. متغیرهای محیطی (Variables) رو تنظیم کن — لیست پایین
5. Generate Domain بزن و آدرس سایتت آماده‌ست — دیتابیس خودش خودکار ساخته می‌شه، نیازی به ترمینال نیست

جزئیات کامل هر قدم پایین اومده 👇

---

## مرحله ۱: آپلود پروژه به گیت‌هاب

### روش A — از طریق وب‌سایت گیت‌هاب (بدون نیاز به git نصب‌شده)

1. برو به [github.com](https://github.com) و وارد حسابت شو
2. روی دکمه سبز **"New"** (یا "+" بالای صفحه → "New repository") بزن
3. اسم بذار: `barber-backend`
4. Public یا Private — هر کدوم فرقی نداره
5. **"Create repository"** رو بزن
6. توی صفحه‌ای که باز می‌شه، دنبال لینک **"uploading an existing file"** بگرد و روش کلیک کن
7. فایل `barber-backend.zip` رو که از من گرفتی، اول **اکسترکت (unzip) کن**:
   - روی موبایل: با اپ‌های مدیریت فایل مثل "Files" (اندروید) یا "ZArchiver" می‌تونی unzip کنی
   - روی کامپیوتر: راست‌کلیک → Extract / استخراج
8. حالا همه فایل‌ها و پوشه‌های داخل `barber-backend` (نه خود فایل zip) رو بکش و بنداز توی باکس آپلود گیت‌هاب
   > ⚠️ نکته: گیت‌هاب پوشه‌های خالی رو قبول نمی‌کنه و آپلود پوشه‌های تودرتو از موبایل گاهی سخته. اگه مشکل داشتی، روش B رو (پایین) استفاده کن.
9. پایین صفحه دکمه سبز **"Commit changes"** رو بزن

### روش B — اگه آپلود مستقیم مشکل داشت (پیشنهادی برای موبایل)

از اپ **GitHub موبایل** (اندروید/iOS) استفاده کن:
1. اپ GitHub رو نصب کن و وارد شو
2. یه ریپازیتوری جدید بساز (`barber-backend`)
3. توی اپ، امکان آپلود فایل تکی داره — برای پروژه‌های چندفایلی بهتره از **Codespaces** گیت‌هاب استفاده کنی (مرحله بعد توضیح میدم) یا از یه کامپیوتر عمومی (کافی‌نت، کتابخونه) یک بار برای آپلود کمک بگیری.

### روش C — راحت‌ترین راه: GitHub Codespaces (کاملاً در مرورگر، مثل یه کامپیوتر آنلاین رایگان)

این بهترین گزینه‌ست چون همه‌چی توی مرورگر انجام می‌شه، حتی روی موبایل:

1. یه ریپازیتوری خالی بساز به اسم `barber-backend` (طبق مرحله ۱ تا ۵ بالا، ولی این بار هیچی آپلود نکن)
2. توی صفحه ریپو، روی دکمه سبز **"Code"** بزن
3. تب **"Codespaces"** رو انتخاب کن
4. **"Create codespace on main"** رو بزن — چند ثانیه صبر کن تا یه محیط VS Code کامل توی مرورگرت باز بشه (حتی روی موبایل هم کار می‌کنه، هرچند صفحه کوچیکه)
5. توی این محیط، یه ترمینال هست (پایین صفحه، یا از منو Terminal → New Terminal)
6. فایل zip رو باید یه‌جوری به codespace برسونی — ساده‌ترین راه:
   - فایل `barber-backend.zip` رو آپلود کن جایی مثل Google Drive یا هر لینک دانلود مستقیم
   - توی ترمینال Codespace بزن:
     ```bash
     wget "لینک_دانلود_فایل_زیپ" -O barber-backend.zip
     unzip barber-backend.zip
     mv barber-backend/* barber-backend/.* . 2>/dev/null
     rm -rf barber-backend barber-backend.zip
     ```
7. حالا توی ترمینال:
   ```bash
   git add .
   git commit -m "اولین آپلود پروژه آرایشگاه"
   git push
   ```
8. تمام! پروژه توی گیت‌هابت هست.

## مرحله ۲: ساخت پروژه روی Railway

1. برو به [railway.app](https://railway.app)
2. با همون حساب گیت‌هابت وارد شو (**"Login with GitHub"**)
3. روی **"New Project"** بزن
4. **"Deploy from GitHub repo"** رو انتخاب کن
5. ریپازیتوری `barber-backend` رو انتخاب کن
6. Railway خودش تشخیص می‌ده Node.js هست و شروع به build می‌کنه

---

## مرحله ۳: اضافه کردن دیتای دائمی (Volume)

چون SQLite یه فایل روی دیسکه، باید یه "volume" دائمی بهش وصل کنیم تا با هر دیپلوی جدید پاک نشه:

1. توی پروژه Railway، روی سرویس (باکس) پروژه‌ات کلیک کن
2. برو به تب **"Settings"**
3. پایین صفحه دنبال بخش **"Volumes"** بگرد
4. **"New Volume"** رو بزن
5. Mount path رو بذار: `/app/data`
6. Save کن

---

## مرحله ۴: تنظیم متغیرهای محیطی (Environment Variables)

1. توی همون پروژه Railway، برو به تب **"Variables"**
2. این متغیرها رو یکی‌یکی اضافه کن (دکمه "New Variable"):

| Key | Value |
|---|---|
| `NODE_ENV` | `production` |
| `PORT` | `3000` |
| `DB_PATH` | `/app/data/barber.db` |
| `JWT_SECRET` | یه رشته رندوم طولانی (پایین توضیح میدم چطور بسازی) |
| `ADMIN_PIN` | یه پین ۴ رقمی دلخواه، نه `1234` |
| `UPLOAD_DIR` | `/app/data/uploads` |
| `SMS_PROVIDER` | `console` (فعلاً، تا SMS واقعی نگرفتی) |
| `ZARINPAL_SANDBOX` | `1` (فعلاً حالت تست) |

### برای ساخت `JWT_SECRET` رندوم:
یکی از این سایت‌ها رو با گوشیت باز کن و یه رشته رندوم ۴۸ کاراکتری کپی کن:
- https://www.random.org/strings/ (طول ۴۸، فقط حروف و عدد)
- یا هر پسورد جنریتور آنلاین با طول بالا

## مرحله ۵: دیتابیس خودکار می‌سازه — کاری لازم نیست! ✅

خبر خوب: پروژه رو طوری تنظیم کردم که **اولین بار که سرور روی Railway بالا میاد، خودش تشخیص می‌ده دیتابیس خالیه و خودکار همه‌چیز رو می‌سازه** (جدول‌ها + کارمندان نمونه + سرویس‌ها + کوپن‌ها). نیازی به دسترسی به ترمینال یا Shell نداری — این مرحله رو رد کن و برو مرحله بعد.

اگه خواستی توی لاگ‌های Railway (تب Deployments → آخرین دیپلوی) این خط رو ببینی که تأیید کنه کار کرده:
```
🌱 دیتابیس خالیه — در حال ساخت داده‌های نمونه...
✅ Seed completed!
```

## مرحله ۶: گرفتن دامنه رایگان Railway

1. توی تب **"Settings"** سرویس، بخش **"Networking"** یا **"Domains"** رو پیدا کن
2. **"Generate Domain"** رو بزن
3. یه آدرس مثل `barber-backend-production.up.railway.app` بهت میده

همینه! سایتت زنده‌ست روی:
```
https://barber-backend-production.up.railway.app/
```

## مرحله ۷: تست

از گوشی یا کامپیوتر برو به آدرس بالا و باید صفحه اصلی آرایشگاه رو ببینی. برای پنل مدیر:
```
https://آدرس‌ت.up.railway.app/admin.html
```

## ⚠️ نکته درباره OTP و پرداخت

فعلاً چون `SMS_PROVIDER=console` گذاشتیم، کد OTP توی **لاگ‌های Railway** نمایش داده می‌شه (نه پیامک واقعی). برای دیدنش:
1. توی پروژه Railway برو به تب **"Deployments"**
2. روی آخرین دیپلوی کلیک کن
3. لاگ‌ها رو ببین — دنبال خط `📱 [SMS - DEV MODE]` بگرد

همینطور پرداخت هم چون `ZARINPAL_SANDBOX=1` هست، همیشه به صورت خودکار "موفق" برمی‌گرده — یعنی می‌تونی کل فلوی رزرو رو تست کنی بدون درگاه واقعی.
